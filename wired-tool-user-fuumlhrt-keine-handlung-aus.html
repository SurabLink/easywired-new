Wired-Tool: User führt KEINE Handlung aus


​Wired-Tool

WIRED Negative Bedingung: User führt KEINE Handlung aus


​​Daten
Wired-Name: User führt KEINE Handlung aus, kurz: Aktions-bdg (negativ)
Wiredtyp: ​Bedingung
Auslösequelle: Benötigt einen auslösenden Typ (Habbo, Bot, Haustier).

Funktion: Auslösen im Wired-Stapel ist erlaubt, wenn der auslösende Typ eine ausgewählte Handlung NICHT durchführt.

Definition: Zu den Handlungen zählen: Winken, Kuss hauchen, Lachen, Daumen hoch, Aufwachen, Schlafen, Setzen, Aufstehen, Liegen, Schild und Tanzen. Schild und Tanzen können zusätzlich noch nach bestimmten Schild oder Tanz gefiltert werden. Während der Handlung kann der auslösende Typ den Wired-Stapel ständig ausführen.

Erweiterte Einstellungen: "Bedingung stimmt überein wenn" und Userquelle können weiter eingestellt werden.

Verwandte Wireds: User führt eine Handlung durch, User führt Handlung aus
Anwendungsbereich: ​Interaktivitäten, AFK-Bereiche für z. B. AFK-LvL

​
Videobeschreibung: Mit der negativen Bedingung kann eingestellt werden, dass User durch Kollision teleportiert werden, sobald sie nicht wach sind.
