Wired-Tool: Möbeltyp stimmt NICHT überein


​Wired-Tool

WIRED Negative Bedingung: Möbel stimmt NICHT überein

​​Daten
Wired-Name: Möbel stimmt NICHT überein, kurz: Möbeltyp
Wiredtyp: Negative Bedingung
Auslösequelle: Benötigt KEINEN auslösenden Typ (Habbo, Bot, Haustier).​

Funktion: Auslösen im Wired-Stapel ist erlaubt, wenn das "auslösende Möbel" Keines der ausgewählten Möbelstücke innerhalb der Bedingung ist.

Definition: Möbeltyp bezieht sich nicht auf eine Möbelsorte oder einer Möbelkategorie im allgemeinen. Ein Möbel gehört zum selben Möbeltyp, wenn dieser innerhalb einer Möbeltyp-Bedingung gespeichert ist. Anders ausgedrückt: Alle gespeicherten Möbeln innerhalb einer Möbeltyp-Bedingung gehören zum selben Typ. Wenn ein Möbel aus diesem Typ in einem Auslöser-Wired gespeichert ist, wird beim Auslösen mit diesem Möbel die Bedingung nicht erfüllt. Das Auslösen ist erst dann erlaubt, wenn das "auslösende Möbel" keines der ausgewählten Möbel innerhalb der Bedingung ist.  Beim Auslöser Kollision kann kein Möbel ausgewählt werden, daher wird das Möbelstück als "auslösender Möbel" bezeichnet. Die Bedingung Möbel auf Möbel (positiv und negativ) wird ebenfalls berücksichtigt. In diesem Fall können sogar automatische Auslöser wie Effekt wiederholen verwendet werden.

Erweiterte Einstellungen: "Bedingung stimmt überein wenn​", "Das passende Möbel" und "Möbel zum Vergleichen" können weiter eingestellt werden.

Verwandte Wireds: Möbelart stimmt überein
Anwendungsbereich: Kollision mit Ausnahmemöbel, Wired-Minimierungen, ortsunabhängige Möbel-Erkennungen

​
Videobeschreibung: Im Video ist ein Beispiel dazu, wie man eine ortsunabhängige Möbel-Erkennung einstellt. Das kann für Games interessant sein, wo ein bestimmtes Möbel "gefunden" werden soll, mit dem man dann interagiert. So muss nicht jeder Ort dafür einzeln eingestellt sein (z. B. mit Posi-Wireds).
